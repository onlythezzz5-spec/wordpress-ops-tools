# PIX 增强版 - 上传改进说明

## 问题修复

### 1. 上传稳定性改进 ✅

**问题原因**:
- 错误处理不完善
- 没有详细的错误信息
- Transient 限制过严格
- 没有重试机制

**改进方案**:
- ✅ 详细的错误代码和消息（14个错误情况）
- ✅ 放宽速率限制（5个重试，60秒冷却）
- ✅ 多层文件验证（大小、类型、哈希）
- ✅ 附件有效性检查（确保真的保存了）
- ✅ 更好的日志记录

### 2. 后台凭证显示 ✅

**改进内容**:

#### A. 完整的支付信息显示
```
📋 支付状态总览
├─ 订单状态
├─ 支付状态 (待支付/已完成/已过期)
├─ 支付金额
├─ 支付发起时间
├─ 过期时间 (还剩几分钟)
├─ 凭证上传时间
├─ 凭证哈希值
└─ 重试次数
```

#### B. 清晰的凭证显示
- ✅ 显示凭证图片预览
- ✅ 显示文件名
- ✅ 显示上传时间
- ✅ 提供原图下载功能
- ✅ 错误处理（附件不存在时的提示）

#### C. 视觉化的支付状态
- ✅ 绿色徽章：已完成
- ✅ 黄色徽章：待验证
- ✅ 红色徽章：已过期
- ✅ 倒计时显示（还剩多少时间）

### 3. 管理员功能增强 ✅

新增3个管理员操作按钮：

#### ✓ 标记为已支付
- 验证订单是否有凭证
- 更新支付状态为已完成
- 更新订单状态为处理中
- 发送客户确认邮件

#### ⏱ 延长期限
- 可指定延长时长（1-1440分钟）
- 更新数据库中的过期时间
- 记录管理员操作

#### 🗑 删除凭证
- 删除附件文件
- 清除所有相关元数据
- 重置支付状态为待验证
- 允许客户重新上传

## 改进的错误处理

| 错误代码 | 含义 | 用户提示 |
|---------|------|--------|
| RATE_LIMITED | 上传过于频繁 | 请等待1分钟后重试 |
| INVALID_ORDER_ID | 订单ID无效 | 订单ID有问题 |
| ORDER_NOT_FOUND | 订单不存在 | 找不到此订单 |
| NO_FILE | 未选择文件 | 请选择要上传的文件 |
| UPLOAD_ERROR_* | 上传错误 | 详细的上传错误信息 |
| FILE_TOO_LARGE | 文件过大 | 显示实际大小和限制 |
| FILE_TOO_SMALL | 文件太小 | 文件为空或损坏 |
| INVALID_FILE_TYPE | 格式不对 | 只支持 JPG/PNG/PDF |
| INVALID_MIME_TYPE | MIME类型不对 | 检测到的 MIME 类型 |
| FILE_NOT_READABLE | 文件不可读 | 文件访问权限问题 |
| HASH_FAILED | 哈希计算失败 | 文件处理失败 |
| BLACKLISTED | 在黑名单中 | 此凭证被标记为欺诈 |
| DUPLICATE_PROOF | 重复凭证 | 已在订单 #X 中使用 |
| MEDIA_UPLOAD_FAILED | 媒体库上传失败 | WordPress 媒体库错误 |
| ATTACHMENT_INVALID | 附件无效 | 附件未正确保存 |
| ATTACHMENT_URL_FAILED | 生成URL失败 | 无法获取图片 URL |
| ORDER_SAVE_FAILED | 订单保存失败 | 无法保存订单数据 |
| DB_UPDATE_FAILED | 数据库更新失败 | 数据库操作错误 |

## 改进的数据存储

### 新增订单元数据
```php
_pix_proof_attachment        - 附件 ID
_pix_proof_attachment_url    - 附件 URL (新增)
_pix_proof_hash              - 凭证哈希
_pix_proof_filename          - 文件名 (新增)
_pix_proof_upload_time       - 上传时间 (新增)
```

### 数据库记录更详细

支付表现在记录：
- `proof_uploaded_time` - 凭证上传的准确时间
- `proof_hash` - 凭证的 MD5 哈希
- `status` - 支付状态（pending/completed/expired）

## 使用示例

### 场景 1: 客户上传了凭证，但显示错误

**旧流程**: 无法判断什么原因失败  
**新流程**: 
1. 前端显示具体错误（如"文件太大"）
2. 后台管理员可以查看错误日志
3. 后端详细记录所有尝试
4. 允许客户重试

### 场景 2: 管理员发现凭证错误

**操作**:
1. 后台订单页面 → PIX 支付管理
2. 点击"🗑 删除凭证"
3. 凭证被删除，订单状态重置
4. 客户可以重新上传正确的凭证

### 场景 3: 客户没在规定时间内支付

**操作**:
1. 支付倒计时显示还剩时间
2. 时间过期自动标记为过期
3. 管理员可手动延长期限
4. 或让客户创建新订单

## 调试建议

### 查看上传日志

```php
// 在 wp-config.php 中启用调试
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);

// 然后查看: /wp-content/debug.log
tail -f /wp-content/debug.log | grep pix
```

### 检查附件是否正确保存

```sql
-- 查询订单的附件
SELECT * FROM wp_postmeta 
WHERE post_id = [order_id] 
AND meta_key LIKE '%pix_proof%';

-- 查询附件本身
SELECT * FROM wp_posts 
WHERE post_type = 'attachment' 
AND post_parent = [order_id];
```

### 测试上传

1. 前台下单并选择 PIX 支付
2. 上传一个小的 JPG 图片
3. 检查:
   - 前台是否显示成功消息
   - 后台订单页是否显示凭证
   - 数据库中是否有记录
